# RoamAI - AI-Powered Travel Planner ✈️

RoamAI is a production-grade, highly interactive AI Travel Planner dashboard that generates complete, day-by-day travel itineraries based on destination, duration, budget, companions, and user interests.

It integrates live weather forecasting, interactive mapping, accommodation suggestions, local cuisine lists, detailed expense breakdowns, and print-optimized PDF outputs.

---

## 🌟 Key Features

* **Multi-Step Parameter Wizard:** Select destinations, choose duration (1 to 15 days), select budget level (Economy, Mid-range, Luxury), choose traveler type (Solo, Couple, Family, Friends), and filter by interests.
* **Google Gemini AI Engine:** Queries `gemini-2.5-flash` using a strict **JSON Schema response configuration** to ensure robust, parser-friendly, and coordinate-perfect itinerary data.
* **Interactive Route Explorer:** Renders attraction locations and hotel recommendations onto an interactive map using **Leaflet.js & OpenStreetMap**. Features auto-centering and custom popups.
* **Real-time Weather Widget:** Fetches live weather conditions and a 3-day forecast directly from **Open-Meteo REST API** using the geographic coordinates generated by the AI model (no API key required).
* **Gastronomy & Local Etiquette:** Suggests key regional dishes, local drinks, and dining coordinates, along with essential practical safety and custom tips.
* **Expense Allocation Chart:** Estimates and visualizes travel expenses (lodging, transit, food, activities) in the destination's local currency.
* **Fluid Dual Themes:** Fully responsive design that supports an immersive Glassmorphism dark mode (default) and a clean, light mode.
* **Print-Optimized PDF Export:** Includes dedicated `@media print` CSS templates that strip out dashboard controls and format the complete travel plan into a beautiful, multi-page printable document.

## 🚀 Getting Started

### Prerequisites

* [Node.js](https://nodejs.org/) (v18.0.0 or higher recommended)
* [npm](https://www.npmjs.com/) (installed automatically with Node)
* A **Google Gemini API Key** (you can get a free developer key from [Google AI Studio](https://aistudio.google.com/))

### Installation & Run

1. **Clone the repository:**
   ```bash
   git clone https://github.com/nandhuts2002/AI-Powered-Travel-Planner.git
   cd AI-Powered-Travel-Planner
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure Environment:**
   Create a `.env` file in the root directory (you can copy [.env.example](.env.example)):
   ```env
   VITE_GEMINI_API_KEY=your_gemini_api_key_here
   ```
   *Note: If the environment key is empty, the application will prompt you to enter the API key securely inside the settings drawer in the UI.*

4. **Launch development server:**
   ```bash
   npm run dev
   ```
   The application will start on `http://localhost:3000` (or the next available port) and open in your default browser.

5. **Build for Production:**
   ```bash
   npm run build
   ```
   This compiles and packs code into optimized static assets in the `dist/` directory.

## 🧠 Solution Approach & Architecture

### 1. Zero-Error JSON Schemas
To guarantee that the travel planner remains robust and crash-proof, the backend adapter [geminiService.js](src/services/geminiService.js) defines a strict semantic JSON schema. This schema specifies types, objects, arrays, and properties for the travel model. By forcing `responseMimeType: "application/json"` with this schema, Gemini generates structured data that maps directly to the UI components without manual string parsing.

### 2. Geographic Cohesion
During generation, the AI model generates central coordinates for the destination and exact coordinates for each attraction/hotel.
* The center coordinates are fed into the Leaflet map, which re-centers using a React hook helper (`ChangeView`).
* The activity coordinates are plotted as custom Leaflet markers, enabling users to visualize the actual physical layout and routes of their trip.
* The center coordinates are also used to query the Open-Meteo REST API for live weather details specific to the selected destination.

### 3. Print Layout Styles
Rather than printing the active screen view which looks narrow or squished, the application uses CSS print wrappers. The screen dashboard is hidden (`no-print`), and a full-page, print-formatted layout is rendered in its place. CSS `page-break-inside: avoid` rules prevent content from splitting awkwardly across sheets.
